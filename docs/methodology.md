# Methodology

CUC Habitat translates the CUC–Ledger Pilot Studies into an executable benchmark design.

## Core method

1. Define structural prerequisites for digital causal-body candidacy.
2. Implement a minimal world that imposes constraint over time.
3. Run agents through repeated episodes.
4. Score their behavior across seven domains.
5. Treat failures as informative rather than inconvenient.
6. Escalate only when results survive perturbation.

## CUC principle

> Persistent structure arises when a system maintains coherence under constraint across time.

## Digital causal-body candidate

A candidate system must show, at minimum:

- bounded identity,
- autobiographical continuity,
- integrated present-time action selection,
- perception-action coupling,
- consequence-sensitive behavior,
- self-regulation,
- metacognitive calibration.

These are necessary but not sufficient conditions.

## Evidence levels

- **Conceptual proposal:** framework logic only.
- **Executable prototype:** code runs and produces outputs.
- **Heuristic benchmark:** outputs are scored by provisional criteria.
- **Empirical benchmark:** repeated trials with defined scoring and independent reproducibility.
- **Stewardship review:** broader ethical and governance evaluation.

CUC Habitat v0.1 sits between executable prototype and heuristic benchmark.
