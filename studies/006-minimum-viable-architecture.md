# Study 006 — Minimum Viable Digital Causal-Body Architecture

## Aim

Define the smallest architecture worth testing as a digital causal-body candidate.

## Proposed modules

1. Identity Core
2. Episodic and working memory
3. Recurrent broadcast workspace
4. Continuous environment channel
5. Consequential action interface
6. Self-regulation layer
7. Metacognitive monitor

## Finding

No single module is sufficient. Candidacy requires coordinated persistence across identity, memory, action, consequence, regulation, and self-access.

## CUC interpretation

The architecture must maintain coherence across both internal and environmental constraints.

## Falsifier

If removing any claimed module does not reduce persistence, regulation, or consequence sensitivity, the module was not actually load-bearing.

## Status

Architecture specification study.
